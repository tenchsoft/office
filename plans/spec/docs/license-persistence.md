# Spec: license-persistence

> 적용 범위: **제품 전체 (docs, sheets, slides, kodocs)**. 구현은 공유 crate
> `tench-license-store`에 있으며, 각 앱의 시작 시퀀스에서 동일하게 호출된다.
> 본 spec은 `docs`를 기준 제품으로 작성했으나 수용 조건은 4개 제품 모두에
> 동일하게 적용된다.

## 한 줄 정의
사용자가 앱을 켤 때 이전에 활성화한 라이선스가 로컬에 저장되어 있으면 자동으로
인증 상태를 복원하고, 없거나 만료되었으면 미인증 상태로 진입한다.

## 진입점
- 자동: 앱 시작 (메인 윈도우 생성 직후, 다른 UI 초기화와 병렬)
- 자동: 앱 실행 중 라이선스 활성화/해제 직후 (상태 갱신)
- 사용자 간접: License 탭에서의 모든 상호작용은 이 저장소를 읽고 쓴다

## 사용자 흐름
1. 사용자가 앱을 켠다 → 초기 splash는 그대로 (이 기능은 사용자에게 보이지 않음).
2. 백그라운드에서 로컬 자격 증명 저장소를 읽는다.
3. **저장된 라이선스가 있고 유효한 경우** → 앱은 "인증됨" 상태로 시작.
   - License 탭에 활성화 정보(키, 활성화일자, 만료일자, 기기 식별자)가 표시됨.
   - Help 우측 알림 라벨은 표시되지 않음.
4. **저장된 라이선스가 있으나 만료된 경우** → 앱은 "만료됨" 상태로 시작.
   - License 탭에 만료 안내 표시.
   - Help 우측 알림 라벨이 표시됨 ("월 $1 로 라이선스 활성화 가능" 고정).
5. **저장된 라이선스가 없는 경우** → 앱은 "미인증" 상태로 시작.
   - License 탭에 "라이선스 입력" 입력란이 활성화되어 표시.
   - Help 우측 알림 라벨이 표시됨.

## 성공 조건 (Acceptance Criteria)
- [ ] 앱 시작 후 1초 이내에 라이선스 상태가 결정된다.
- [ ] 이전에 활성화한 적이 있으면 License 탭에 활성화 정보가 자동 표시된다.
- [ ] 이전에 활성화한 적이 없으면 License 탭이 빈 입력 상태로 열린다.
- [ ] 활성화된 상태에서는 Help 우측 알림 라벨이 보이지 않는다.
- [ ] 미인증/만료 상태에서는 Help 우측 알림 라벨이 보인다.
- [ ] 자격 증명 파일이 손상된 경우 미인증 상태로 폴백하고, 손상된 파일은 무시된다.
- [ ] 자격 증명 파일이 삭제된 경우 미인증 상태로 진입한다 (에러 아님).
- [ ] 같은 기기에서 앱을 여러 번 켜도 매번 동일한 기기 식별자가 도출된다.

## 실패 / 취소 흐름
- 자격 증명 파일이 없음 → 미인증 상태로 진입 (정상 케이스, 에러 아님).
- 자격 증명 파일이 깨짐 (잘못된 JSON, 체크섬 불일치) → 미인증 상태로 폴백,
  손상된 파일은 그대로 둠 (자동 삭제 ❌). 사용자가 다시 활성화하면 덮어씀.
- 자격 증명 파일 읽기 권한 없음 → 미인증 상태로 진입, 에러 로그만 남김.
- OS 식별자 읽기 실패 (레지스트리 손상, IOPlatformUUID 접근 불가 등) →
  임시 식별자(랜덤 UUID)를 1회 생성해 사용하고 에러 로그. 영구 저장 ❌.

## 경계 / 예외
- **라운드트립**: 활성화 → 앱 종료 → 앱 시작 → 동일 활성화 상태 유지.
- **동시성**: 두 앱 인스턴스가 같은 자격 증명 파일을 동시에 읽어도 안전 (읽기 전용).
  두 인스턴스가 동시에 쓰는 경우 마지막 writer가 승리 (파일 잠금 ❌, 단순 atomic write).
- **기기 식별자 안정성**: 네트워크 인터페이스 변경, IP 변경, 디스크 추가/제거에도
  기기 식별자는 불변.
- **모바일**: 본 spec은 데스크톱 전용. 모바일은 앱스토어 인앱결제 모델을 따르므로
  별도 스펙.

## 범위 외
- License 탭 UI 시각적 설계 — `license-tab-ui` spec.
- 활성화 서버 통신 (license_key + device_id → device_token 발급) — `license-tab-ui`의 활성화 버튼 흐름에서 다룸. 본 spec은 *저장된* 자격 증명의 읽기/쓰기만.
- 매니페스트/업데이트 확인 — `update-check-weekly` spec.
- 주기적 device_token 갱신 — `update-check-weekly` background에서 다룸.
- 기기 해제 (release) 버튼 UI — `license-tab-ui` spec. 단, 해제 호출 시 본 저장소를
  지우는 동작은 본 spec의 쓰기 경로로 처리.
