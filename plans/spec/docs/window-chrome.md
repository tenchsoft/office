# Spec: window-chrome

> 적용 범위: **제품 전체 (docs, sheets, slides, kodocs)**. 구현은 공유 크레이트
> `tench-ui`에 있으며, 각 앱의 최상단 헤더(메뉴 바 / 툴바)에 동일하게 적용된다.
> 본 spec은 `docs`를 기준 제품으로 작성했으나 수용 조건은 4개 제품 모두에
> 동일하게 적용된다.

## 한 줄 정의
사용자가 창 우측 상단의 캡션 버튼으로 창을 최소화/최대화/닫고, 최상단 헤더 영역의
빈 공간을 드래그해 창을 이동할 수 있다.

## 진입점
- 캡션 버튼: 창 우측 상단의 세 버튼 (좌→우: 최소화, 최대화/복원, 닫기)
- 드래그 이동: 최상단 헤더(메뉴 바 또는 툴바)에서 **빈 영역**을 누르고 드래그
- 단축키: 없음 (OS 단축키 Alt+F4 등은 그대로 동작)

## 사용자 흐름
1. 사용자가 창을 엶 → 우측 상단에 세 캡션 버튼이 보인다 (OS 제목표시줄은 없음).
2. 사용자가 **최소화** 버튼을 클릭 → 창이 작업표시줄로 내려간다.
3. 사용자가 작업표시줄에서 창을 복원 → 다시 보인다.
4. 사용자가 **최대화** 버튼을 클릭 → 창이 전체 화면으로 커지고, 버튼 아이콘이
   "복원" 형태(겹친 두 사각형)로 바뀐다.
5. 사용자가 다시 **복원** 버튼을 클릭 → 이전 크기로 돌아가고, 아이콘이 "최대화"
   형태(단일 사각형)로 바뀐다.
6. 사용자가 헤더의 **빈 영역**(메뉴 항목과 캡션 버튼 사이)을 누르고 드래그 →
   창이 커서를 따라 이동한다.
7. 사용자가 **닫기** 버튼을 클릭 → 창이 닫히고 앱이 종료된다.

## 성공 조건 (Acceptance Criteria)
- [ ] OS 제목표시줄 없이 창이 열린다 (`decorations` off).
- [ ] 우측 상단에 최소화 / 최대화 / 닫기 버튼 3개가 보인다.
- [ ] 최소화 클릭 시 창이 작업표시줄로 내려간다.
- [ ] 최대화 클릭 시 창이 최대화되고 버튼 아이콘이 "복원"으로 바뀐다.
- [ ] 복원 상태에서 다시 클릭 시 이전 크기로 돌아가고 아이콘이 "최대화"로 바뀐다.
- [ ] 닫기 클릭 시 창이 닫히고 앱이 종료된다.
- [ ] 헤더 빈 영역 드래그 시 창이 이동한다.
- [ ] 헤더의 메뉴 항목 / 툴바 버튼 클릭은 여전히 정상 동작한다 (드래그로 가로채지 않음).
- [ ] 캡션 버튼 hover 시 시각적 피드백(배경색, 닫기 버튼은 빨강)이 있다.
- [ ] 기존 헤더 우측 콘텐츠(제목, 저장 상태 등)가 캡션 버튼 영역과 겹치지 않는다.

## 실패 / 취소 흐름
- 메뉴가 열려 있을 때 헤더 빈 영역 클릭 → 메뉴가 닫힌다 (드래그는 시작 안 함).
- 모달 활성 중 캡션 버튼 클릭 → 캡션 동작이 우선한다 (모달보다 상위 레이어).
- 최대화 불가 상태 (일부 플랫폼) → 최대화 버튼 무동작 또는 무해한 no-op.

## 경계 / 예외
- 라운드트립: 최대화 → 복원 → 최대화 반복 시 아이콘이 일관되게 토글.
- 빈 헤더 클릭 후 드래그 없이 놓기 → 창 이동 없음 (단순 클릭).
- 모바일: 캡션 버튼은 데스크톱 전용 개념; 모바일 레이아웃에서는 별도 처리 (본 spec 범위 외).

## 범위 외
- 커스텀 타이틀바 위젯(제목 표시, 탭 통합 등) — 별도 spec.
- 모바일/태블릿용 창 컨트롤 — 별도 spec.
- Tauri 번들 경로(`lib.rs::run()`)의 WindowAction 실행 — 네이티브 경로만 본 spec. Tauri 백엔드 drain은 후속.
